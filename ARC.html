对象类型修饰符一共有4种

这些修饰符自动变量初始化为nil 1.__strong 强引用修饰符 也是对象类型默认修饰符,强引用修饰符持有对象实例,随着强引用的失效, 引用的对象会随之释放

2.__weak 弱引用修饰符 不持有对象实例, 弱引用的对象被废弃,则此引用将自动失效且处于nil状态

3._unsafeunretained 不安全修饰符,该修饰符的变量不属于编译器的内存管理对象

4.autoreleasing修饰符 在ARC有效时候用@autoreleasepool块替代NSAutoreleasePool类, 用附有autoreleasing修饰符的变量替代autorelease方法(为了自动注册到Pool) id _weak obj1 = obj0; id _autoreleasing tmp = obj1; (确保@autoreleasepool块结束之前确保该对象存在)

所谓内存泄露就是应当废弃的对象在超出其生存周期后继续存在

ARC规则: 1.ARC有效时, 不能使用retain/release/retainCount/autorelease 不能使用NSAllocateObject/NSDeallocateObject

2.命名规则：以init开始的方法必须是实例方法并且必须返回对象

3.无论ARC是否有效, 不要显式调用dealloc

4.使用@autoreleasepool块替代NSAutoreleasePool

5.不能使用区域(NSZone)

6.对象型变量不能作为C语言结构体的成员 struct Data{ NSMutableArray _unsafeunretained *array; };

7.id和void *的显式转换

悬垂指针: 指向曾经存在的对象，但该对象已经不再存在了，此类指针称为悬垂指针。结果未定义，往往导致程序错误，而且难以检测

可以通过bridge把id和void*相互转换(不安全) void *p = (bridge void *)obj; id o = (__bridge id)p;

_bridgeretained 与retain相似 _bridgetransfer 与release相似

分配内存推荐用calloc这样的函数分配,分配后置为nil calloc = malloc + memset
